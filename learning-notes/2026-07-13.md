## 编译输出的判断方法

今天看到的主要 warning 包括：

- `argument unused during compilation`
- `invalid offsetof`
- 第三方库中的未使用函数和兼容性提示

这些 warning 没有导致本次构建失败。判断真实失败时，应优先在“输出 → 生成”中寻找第一条：

```text
FAILED
error
fatal error
ninja: build stopped
```

不要只根据错误列表数量或红色波浪线判断构建是否失败。

## IntelliSense 与编译器

IntelliSense 是 Visual Studio 的编辑器分析系统，负责自动补全、跳转、参数提示和红色波浪线。它类似 IntelliJ IDEA 的索引与实时检查。

真正决定程序能否构建的是：

```text
CMake → Ninja → clang-cl → 链接器
```

因此可能出现“IntelliSense 标红，但 Clang 实际构建成功”。本次就属于这种情况。

## `argc` 与 `argv`

C++ 程序入口：

```cpp
int main(int argc, char* argv[])
```

- `argc`：命令行参数数量。
- `argv`：C 风格字符串数组。

本次启动命令相当于：

```text
shadps4.exe --help
```

调试器中观察到：

```text
argc = 2
argv[0] = E:\ps4\shadPS4\Build\x64-Clang-Debug\shadps4.exe
argv[1] = --help
```

与 Java 的 `main(String[] args)` 不同，C++ 的 `argv[0]` 通常包含程序自身路径。

## 第一次断点调试

断点位置：

```cpp
app.parse(argc, argv);
```

执行流程：

```text
--help
  ↓
app.parse(argc, argv)
  ↓
CLI::CallForHelp
  ↓
catch (const CLI::ParseError& e)
  ↓
app.exit(e)
  ↓
进程以退出码 0 正常结束
```

调试器中观察到：

```text
actual_exit_code = 0
error_name = "CallForHelp"
```

`CallForHelp` 虽然通过 C++ 异常传递，但它表示用户请求帮助，不是程序故障。
